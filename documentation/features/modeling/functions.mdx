<Warning>
  Your [number format](https://docs.francis.app/documentation/getting-started/account/preferences) in Francis determines whether you'll use commas (US) or semicolons (EU) to separate function parameters. All examples below use the US format with commas.
</Warning>

<AccordionGroup>
  <Accordion title='avg("row_name"[range_start:range_end])'>
    The `avg()` function calculates the average of the values in the specified time series from `range_start` to `range_end`.

    ```typescript
    avg("Revenue"[-3:0])
    ```
  </Accordion>
  <Accordion title='avg_ytd("row_name")'>
    The `avg_ytd()` function returns the average of the values in a time series from the beginning of the fiscal year until and including the current month.

    ```typescript
    avg_ytd("Revenue")
    ```
  </Accordion>
  <Accordion title='avg_last_year("row_name")'>
    The `avg_last_year()` function returns the average of the values in the previous fiscal year. This function is influenced by your fiscal year settings.

    ```typescript
    avg_last_year("Revenue")
    ```
  </Accordion>
  <Accordion title='avg_last("row_name", num_months)'>
    The `avg_last()` function returns the average of the values in the time series going back _n_ months, as defined by the `num_months` parameter.

    ```typescript
    avg_last("Revenue", 6)
    ```
  </Accordion>
  <Accordion title='median("row_name"[range_start:range_end])'>
    The `median()` function returns the median of the values in the specified time series from `range_start` to `range_end`

    ```typescript
    median("Revenue"[-3:0])
    ```
  </Accordion>
  <Accordion title='median_ytd("row_name")'>
    The `median_ytd()` function returns the median of the values in a time series from the beginning of the fiscal year until and including the current month.

    ```typescript

    ```
  </Accordion>
  <Accordion title='median_last_year("row_name")'>
    The `median_last_year()` function returns the median of the values in the previous fiscal year. This function is influenced by your fiscal year settings.

    ```typescript
    median_last_year("Revenue")
    ```
  </Accordion>
  <Accordion title='median_last("row_name", num_months)'>
    The `median_last()` function returns the median of the values in the time series going back _X_ months, as defined by the `num_months` parameter.

    ```typescript
    median_last("Revenue", 6)
    ```
  </Accordion>
  <Accordion title='sum("row_name"[range_start:range_end])'>
    The `sum()` function calculates the total of the values in the specified time series from `range_start` to `range_end`.

    ```typescript
    sum("Revenue"[-3:0])
    ```
  </Accordion>
  <Accordion title='sum_ytd("row_name")'>
    The `sum_ytd()` function returns the sum of the values in a time series from the beginning of the fiscal year until and including the current month.

    ```typescript
    sum_ytd("Revenue")
    ```
  </Accordion>
  <Accordion title='sum_last_year("row_name")'>
    The `sum_last_year()` function returns the sum of the values in the previous fiscal year. This function is influenced by your fiscal year settings.

  </Accordion>
  <Accordion title='sum_last("row_name", num_months)'>
    The `sum_last()` function returns the sum of the values in the time series going back _X_ months, as defined by the `num_months` parameter.

  </Accordion>
  <Accordion title='receivables("row_name", payment_days)'>
    The `receivables()` function accumulates revenue amounts from previous months based on an assumption of payment days. Based on the number of payment days, the function loops back through previous months to collect revenue as part of receivables.

    The function assumes 30 days per month. The payment days input must be hardcoded and cannot reference a cell.

    ```typescript
    receivables("Revenue", 30)
    ```
  </Accordion>
  <Accordion title="ignore_div_zero(a/b)">
    The `ignore_div_zero()` returns a zero if the function's output is a #DIV\!/0 error. The function is used for margin and percentage calculations, where the denominator is occasionally zero.

    ```typescript
    ignore_div_zero("Direct cost"[0] / "Revenue"[0])
    ```
  </Accordion>
  <Accordion title="power(num, exponent)">
    The `power()` function is a mathematical function that returns the value of the parameter _num_ to the power of the _exponent_ parameter.

    ```typescript
    power(10, 2) // returns 100
    ```
  </Accordion>
  <Accordion title="min(a,b)">
    The `min()` function returns the smallest numeric value of the input values.

    ```typescript
    min(5, 10) // returns 5
    ```
  </Accordion>
  <Accordion title="max(a,b)">
    The `max()` function returns the largest numeric value of the input values.

    ```typescript
    max(5, 10) // returns 10
    ```
  </Accordion>
  <Accordion title="abs(num)">
    The `abs()` function returns the absolute value of a number.

    ```typescript
    abs(-10) // returns 10
    ```
  </Accordion>
  <Accordion title="round(num, decimals)">
    The `round()` function rounds the number value up or down to the nearest integer. Optionally, provide desired number of decimals.

    ```typescript
    round(10.4386) // returns 10
    ```
  </Accordion>
  <Accordion title="round_down(num)">
    The `round_down()` function rounds numbers _down_ to the nearest integer.

    ```typescript
    round_down(10.6) // returns 10
    ```
  </Accordion>
  <Accordion title="round_up(num)">
    The `round_up()` function rounds numbers _up_ to the nearest integer.

    ```typescript
    round_up(10.6) // returns 11
    ```
  </Accordion>
  <Accordion title="if(condition, value_if_true, value_if_false)">
    The `if()` function allows you to create a conditional calculation, similar to what is available in Excel or Google Sheets.

    ```typescript
    if("Revenue"[0]=0, 0, "Gross profit"[0]/"Revenue"[0])
    
    // If revenue is zero, display zero; otherwise, calculate the gross profit margin. This specific example prevents #ERRs from dividing by zero.
    ```
  </Accordion>
  <Accordion title="if_month(a,b)">
    The `if_month` function returns `1` if the time period is equal to the month number input in the formula, else `0`.

    - `if_month(1)` returns `1` if the month is January, else `0`
    - `if_month(1, 6)` returns `1` if the month is January or June, else `0`
    - `if_month(1, 6, 12)` returns `1` if the month is January, June, or December, else `0`
  </Accordion>
  <Accordion title="if_quarter_month(num)">
    The `if_quarter_month` function returns `1` if the time period is equal to the month number inside a given quarter. E.g., you can input [1;3] into the formula.

    - `if_quarter_month(1)` returns `1` if the month is January, April, July, or October
    - `if_quarter_month(2)` returns `1` if the month is February, May, August, or November
    - `if_quarter_month(3)` returns `1` if the month is March, June, September, or December
  </Accordion>
  <Accordion title="weekdays()">
    The `weekdays()` function returns the number of weekdays (Monday through Friday) in a given month.

    ```typescript
    weekdays() // January 2025 returns 23
    ```
  </Accordion>
</AccordionGroup>
